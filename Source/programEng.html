Coding

Our coding system is more like Visual Basic. Really, i chose my favorite Pascal and C++ commands. We have a simple educational script language.
Examples folder is waiting for you to be explored.

A few Tooltips:
# You don't have to define variables and variable kinds. All the variables are global. You can use space, number starting names, special characters, however i don't advice. It's assumed that user is aware of mistakes.
# Number (integer and real), string and boolean (True, False) are converted automatically. 1E3 can be used as 1000. Comma "," is used for decimal separator.
# "=" is used for comparison and expression assignment. There is no ";" at line ends. I am thinking of using ";" for multiple line commands.
# "+", "-", "*", "/", "&" (string combination), "^" (exp - exponential), “\” (div - integer division), "%" (mod - modulation), "@" (logical and), "~" (logical or) exist. You can use brackets.
# After variables and components properties, you can use "++" and "--" to increase or to decrease. a++ is an example, however it is not possible ++a for now. a++ is a little faster then a=a+1.
# Unary calculations can be used as a+=2 (equivalent is a=a+2) for all operators (+, -, *, /, ^, %, &, \, @, ~). Do not use space between operator and equal sign like a+ =1.
# Calculation direction is from left to right. There is no priorty for * (multiply) or / (divide). But you can use parenthesis.
a=5+6*3-2 ==> a=21 //normally; * (multiply) is done first, then the rest is done.
a=5+6*3-2 ==> a=31 //in Yunus; 5+6, then 11*3, then 33-2... so a=5+(6*3)-2 is better.
# sin(real), cos(real), random(integer), int(real), not(logical), len(string), lower(string), upper(string), chr(integer), asc(character), delete(string, first, count), mid(string, first, count), pos(string, character), and(integer, integer), or(integer, integer), xor(integer, integer), shr(integer, count), shl(integer, count), time() and date() functions are available. I did not add these to customize window.
# Strings and boolean ("True", "False") are needed to be between " (double quote).
# Commands, component properties and variables are not case sensitive.
# If is used as one or multiple comparison expression. You can use "@" as AND or "~" as OR.
# If's comparison may contain "=" ">" "<" "<>" "<=" ">=". One within another If (Else) blocks work fine.
# iif condition, true line, false line is like in VB. You can't use blocks in true or false part.
# For varName=StartVal, LastVal, Increment is available and can be used one within another.
# While is for rotation command blocks until false comparison. Its finishing word is Do.
# You can write your own Function without parameters.
# "//" is for comment lines. We can do inline comments.
 # Goto is for branching any labeled line.
# You can Tab and put spaces between commands. But Goto's labels and ":" must be adjacent like "error:".
# Canvas is used for text, line, point, rectangle (square), circle (ellipse), PolyLine on Form. Refresh is to clean. These drawings are not persistent.
# playSound filename is for *.wav music files.
# Table component is to use *.xml (Extensible Markup Language) and *.cds (Client Data Set) files to record simple things. SQL (Structured Query Language) support and table design window (shortcut: Ctrl+D) in not finished.
# ExcelTable is for using Excel files. Any cells can be reached and modified, then that file can be saved. New file can not be created.
# BenchmarkStart and BenchmarkEnd is for miliseconds speed calculation of that code block. Its result is added to Error Window.
# command oneLineCommand is for Yunus' own one line command execution. For example;
command a=1
# inport(address) and outport(address, byteValue) is for hardware port usage. For example 888 (hexadecimal 378) is printer's data port address in decimal number system.
